Overview

Problem Formulation

An autonomous indoor service robot operates in a building represented as a grid-based map. It must navigate rooms and corridors, collect resources, avoid obstacles, reach a designated goal, and manage limited energy.

1
Real-World Mapping

Grid cells represent rooms and corridors. Walls model physical barriers. Resources stand in for packages, food, or data points. The goal represents a charging station or exit.

2
Core Challenges

The agent must sense its surroundings, avoid or interact with obstacles, make decisions under uncertainty, and optimise objectives such as time, energy, and reward.

3
Progressive Complexity

Three phases build from reactive movement to memory-guided planning to hierarchical decision-making with stochastic actions and energy constraints.

Research Phases

Three Phases of Development

Each phase introduces new capabilities, building from simple reflexes to intelligent planning.

01

Reactive Navigation

Baseline agent navigating with deterministic rules and random movement.

  • Discrete grid representation (5x5, 6x4)
  • Explicit agent position tracking
  • Three boundary modes: wall, bouncy, wrap
  • Random direction selection
  • Step and coverage statistics
How do different boundary conditions affect agent reachability and navigation efficiency?
02

Object Interaction & Planning

Memory-guided exploration with obstacle pushing and MST-based planning.

  • OOP-based agent design (inheritance)
  • Visited-cell memory and frontier tracking
  • Movable obstacle interaction (push)
  • Candidate edge exploration strategy
  • MST agent using Prim's algorithm
How does agent memory improve navigation efficiency and decision quality?
03

Perception, Hierarchy & Learning

Hierarchical controllers with energy management under stochastic uncertainty.

  • Perception-action loop with enhanced sensing
  • Four controller strategies compared
  • Two-level hierarchical decision-making
  • Stochastic action noise (0% - 30%)
  • Energy budget with depletion and restoration
What is the impact of hierarchical decision-making on agent performance under uncertainty?
Implementation

Key Implementation Details

Step-by-step walkthrough of the core algorithms and design decisions.

Step 1 — Grid Environment

A 2D grid of CellType enums with configurable dimensions and boundary modes. The environment owns all state: agent position, resource locations, obstacle positions. Three boundary handlers determine what happens when the agent tries to step off the edge.

Boundary Logic # Wall mode: reject out-of-bounds moves if mode == 'wall': return (False, current_pos, 'blocked') # Bouncy mode: reverse direction on boundary if mode == 'bouncy': new_dir = opposite(direction) # Wrap mode: toroidal arithmetic if mode == 'wrap': nr, nc = nr % rows, nc % cols

Step 2 — Reactive Agent

The simplest agent. Each step it senses adjacent cells, filters for valid (non-wall) directions, and picks one at random. It records visited cells, action counts, and resource pickups for later analysis.

Step 3 — Memory Agent with Candidate Edges

Maintains a visited set and a candidate edges frontier. On each move, it prefers to step into an unvisited neighbour. When all neighbours are visited, it uses BFS to navigate back to the nearest frontier cell, ensuring systematic coverage instead of aimless wandering.

Step 4 — MST Planning Agent

Before moving, computes a Minimum Spanning Tree over all resource positions plus the goal using Prim's algorithm with Manhattan-distance edge weights. A DFS traversal of the MST produces an efficient visitation order. The agent follows this plan using BFS grid pathfinding between nodes.

Step 5 — Four Controllers

Controller objects are plugged into the hierarchical agent, cleanly separating strategy from mechanics:

ControllerStrategy
ExplorerPicks from all 4 directions (may hit walls)
ReactivePicks from valid directions only
DeliberativeBFS shortest path to goal
HierarchicalTwo-level: strategy selector + navigator

Step 6 — Hierarchical Decision-Making

The high level selects a sub-goal based on the agent's state. If energy is low, it switches to survival mode and heads for the goal. If resources are within reach and energy permits, it enters gather mode. Otherwise, it explores unvisited cells. The low level navigates toward the chosen sub-goal using BFS with greedy fallback.

Step 7 — Stochastic Noise & Energy

Each action has a configurable probability of deviating to a perpendicular direction, simulating real actuator imprecision. Energy depletes by 1 per step and is partially restored (+10) when collecting a resource. The episode ends if energy reaches zero before the goal is reached.

Step 8 — Experiments & Plots

Seven experiments run 50 episodes each with deterministic seeding. Metrics include coverage, goal rate, revisit ratio, resources collected, energy remaining, and total reward. Ten matplotlib plots are generated automatically in the results/ directory.

Phase 1: Boundary Mode Comparison

Figure 1 — Grid Coverage by Boundary Mode. Mean coverage across 50 episodes on the 5x5 grid. Wall mode confines the agent and produces the highest local coverage (78.7%). Bouncy mode is slightly lower (75.5%) because reflections occasionally push the agent away from productive directions. Wrap-around mode drops to 45.6% because the toroidal topology scatters the agent across distant regions rather than filling in nearby gaps. Error bars show one standard deviation.
Figure 2 — Coverage Over Time. Average coverage plotted step-by-step for each boundary mode. Wall and bouncy modes show a steady upward curve that begins to plateau around step 100 as fewer unvisited cells remain. Wrap-around coverage plateaus much earlier at roughly 45%, confirming that the toroidal short-cuts cause the agent to revisit distant regions instead of methodically exploring its neighbourhood.
Figure 3 — Performance by Grid Size. Coverage and goal-reached rate compared between the 5x5 and 6x4 grids using wall boundaries. The 6x4 layout's narrower corridors funnel the random agent toward the goal more reliably (92% vs 72%), while overall coverage is comparable between both layouts (79–81%).

Phase 2: Memory and Planning

Figure 4 — Resources Collected and Goal Rate by Agent Type. Both Memory and MST agents reach the goal in 100% of episodes compared to only 84% for the Reactive agent. The Reactive agent collects slightly more resources on average (2.2) because it wanders longer and stumbles onto them, but at the cost of far more steps and lower reliability.
Figure 5 — Revisit Ratio by Agent Type. The Reactive agent wastes 80.6% of its moves revisiting cells it has already visited. Memory-based agents reduce this to under 17%, a five-fold improvement in navigation efficiency that clearly demonstrates the value of maintaining a visited-cell frontier and candidate-edge exploration strategy.
Figure 6 — Obstacle Impact on Memory Agent. With zero obstacles the Memory agent reaches the goal in 100% of episodes using only ~15 steps. Adding 2 movable obstacles causes steps to spike to ~248 and the goal rate to drop to 18%. With 4 obstacles the situation deteriorates further (8% goal rate), as pushed obstacles frequently block the only available paths through the grid.

Conclusion

Key Takeaways

Summary of findings linking all three research questions to experimental evidence.

1
Boundary Conditions Matter

Wall mode yields the highest coverage (79%) for random agents. Wrap-around creates topological shortcuts that guarantee goal attainment at the cost of thorough exploration.

2
Memory Eliminates Waste

Agent memory reduces the revisit ratio from 81% to 16%, a five-fold improvement. The MST agent guarantees efficient resource collection through global planning.

3
Hierarchy Enables Robustness

The hierarchical controller achieves 100% goal rate and maximum reward across all noise levels and energy budgets. Strategy switching between gather, explore, and survive modes is the key differentiator.

Future Work

Reinforcement learning for adaptive threshold tuning. Multi-agent coordination for collaborative resource gathering. Larger, procedurally generated maps to stress-test scalability. A* heuristic pathfinding for efficiency on bigger grids.
